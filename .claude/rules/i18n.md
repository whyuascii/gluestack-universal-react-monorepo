# Internationalization Rules

Applies to: `packages/i18n/**`

- English (`en/`) is the source of truth â€” add keys here FIRST
- Then add to ALL other languages (currently `es/`) with same keys
- Never translate: JSON keys, `{{placeholder}}` names, array structure
- Namespace files map to features: `auth.json`, `common.json`, `dashboard.json`, etc.
- New namespace? Register in `languages.json` AND both `i18n.web.ts` / `i18n.mobile.ts`
- API errors use i18n keys: `throwError("FORBIDDEN", "errors.forbidden.noPermission")`
- Internal logs stay English: `console.log`, analytics events, database fields
